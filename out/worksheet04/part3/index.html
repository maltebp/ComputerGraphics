In both part 1 and 2 all cubes are rendered to the screen by using model, view and projection matrices.


Model
In both parts, the model matrix for each cube is created by using the Cube's size and position. The matrix is created from these values by using the following expression: mult(translate(position), scalem(size, size, size)) Which basically means that a scaling matrix S and translation matrix T are multiplied as: M = T * S

View
In each part each cube is drawed using an instance of the LookAtCamera class. This class defines a View matrix as a lookat matrix, which translates a world coordinates to view coordinates using a camera target (look at) and camera eye. Thus the view matrix is created using the MV.js lookAt function: V = lookAt(pos, target, up) The camera orientation towards the cube is changed in each cube rendering by primarily changing the position of the camera (pos).


Projection
The projection matrix differs between Part 1 and 2, and is defined by two different "camere" classes deriving from the LookAtCamera class.
In the Part 1 the projection matrix is an orthogonal matrix, which is created by: P = ortho(-width/2, width/2, -height/2, height/2.0, 0, 10000) This matrix moves the origin of the view to the center of the screen (so the look at target is centered), and clips everything below the 0'th z coordinate (in view space).

In Part 2 the projection matrix is a perspective matrix instead, which is defined by: P = perspective(fov, width/height, 0.01, 10000); This adds perspective to the view.


Total matrix
Each cube is rendered by multiplying the cube vertices by the concatination of the above mentioned matrices as follows: transformedVertex = P * V * M * vertex
