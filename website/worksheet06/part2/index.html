Linear vs Nearest

'Nearest' filtering causes pixelation of the texture when its scaled, where as 'linear' blurs the transition between pixels. The linear filter can give smoother scaling, but only comes off realistic, if the texture doesn't contain too hard color transitions that we want to maintain when scaling, like this checkerboard texture. For minification, the linear transition works quite well regardless of the texture.

Mipmapping

Mipmapping allows to make a better estimate of a sized down versions of the texture (i.e. when zooming, or viewing it at a very sharp angle). Without mipmapping, we get this very noisy, pixelated texture when scaled down. Picking the mipmap can also be done linearly or with nearest, and it has the same effect as when picking texels. With the nearest mipmap, we get a very hard transition between the different mipmap levels, which in this case results in block, single color texture in some parts.
The mipmapping can be combined with texel filtering, as texels are picked using either the nearest or linear filter, after the correct mipmap has been picked.
